要求支持常见安卓平台、鸿蒙、鸿蒙next。

## 1. 产品目标与范围

### 1.1 核心目标

构建一款“开始背诗后**全程语音对话**完成背诵”的 App，支持：

* **Android**
* **HarmonyOS**
* **HarmonyOS NEXT**

### 1.2 核心交互要求

* 主界面只有少量按钮/设置项：**“开始背诗”**为唯一必需点击入口。
* 用户点击“开始背诗”后：

  * 进入背诗环节（Session）
  * 后续交互**全部用语音**（提示、确认、纠错、提示字、结束等）
* 输入：ASR **仅支持普通话**，但要容忍：

  * 轻微停顿
  * 重复口头禅/叠词（“嗯…我要背…我要背…”）
* 输出：TTS 支持多种个性化音色（自然、有亲和力），可切换。
* 语言：仅中文（简/繁随系统）；不做英文 UI。

## 2. 功能清单（MVP 必须 / 可选增强）

### 2.1 MVP 必须

1. **开始背诗按钮**：点击进入语音对话 Session
2. **语音识别诗名**：

   * 用户说：“我要背xx诗 / 我要背《xx》 / xx”
   * 有停顿与重复也能识别出诗名（从话语中抽取候选）
3. **识别诗名后引导**：

   * 语音提醒用户先说：**朝代** + **作者**
   * 朝代允许模糊映射（见 6.2）
   * 作者允许：名/字/号/别称（可配置常见说法提示）
4. **背诵校验**：

   * 支持常见发音差异（前后鼻音等）作为“可接受偏差”
   * **声调差异**默认不当作错误，但可选择“提醒正确发音”
   * 同一句允许多个“历史认可的说法”（variant）不判错
     例：“豫章故郡”说成“南昌故郡”也算对（配置）
5. **卡顿提示**：

   * 用户长停顿时，询问“要不要提示？”
   * 提示方式三选一（或组合）：
     a) “开头一个字是 …”
     b) “下一个字是 …”
     c) “大体意思是 …”
6. **完成表扬 + 下一首**：

   * 表扬需**个性化随机**，不能每次同一句
   * 询问是否背下一首
7. **超时自动退出**：

   * 长时间未给出明确诗名或相关指令，自动结束 Session 回到主界面

### 2.2 可选增强（后续迭代）

* 离线包/在线更新诗库（非本次必须，但预留接口）
* 背诵历史与熟练度
* 自定义提示强度、口音容忍度
* 诗句分段策略（按句/按联/按段落）

## 3. 跨平台工程总架构（建议：共享核心 + 平台语音适配）

### 3.1 总体拆分

* **Shared Core（跨平台共享逻辑）**

  * 诗库与索引（Poem DB）
  * NLU（指令/诗名/作者/朝代抽取）
  * 背诵对齐与容错评分（Recite Matcher）
  * Session 状态机（Conversation FSM）
  * 配置与个性化（Config/Persona）
* **Platform Shell（各平台 UI + 语音能力）**

  * UI：开始按钮、设置页、权限页
  * ASR：录音、端点检测、返回文本/置信度/分段
  * TTS：音色列表、播放队列、中断/暂停
  * 权限与系统语言适配

### 3.2 技术实现建议（Codex 生成时按此落工程）

* Shared Core：**纯 TypeScript**（逻辑易迁移到 Android/Harmony；Harmony NEXT 原生 ArkTS 友好）

  * Android：用 Kotlin + JS 引擎（QuickJS / J2V8）或直接用 Kotlin 复刻同等逻辑
  * Harmony/Harmony NEXT：ArkTS 直接复用 TS 逻辑（或复制为 ArkTS）
* 若你更倾向原生统一：Shared Core 也可用 **Rust**（通过 FFI），但生成成本更高。**本方案默认 TS/ArkTS 方案。**

> Codex 实现时：优先把 Shared Core 写成**无平台依赖**的纯逻辑库；平台侧只负责“把语音转文本、把文本交给核心、把核心输出的要说的话交给 TTS”。

## 5. 语音对话 Session 状态机（FSM，必须照做）

### 5.1 状态定义

* `IDLE`：主界面
* `SESSION_START`：播报欢迎语 + 提示说诗名
* `WAIT_POEM_NAME`：等待用户说“要背什么”
* `CONFIRM_POEM_CANDIDATE`：若多候选/低置信度，进行语音澄清
* `WAIT_DYNASTY_AUTHOR`：提示用户说朝代和作者（可分两轮）
* `RECITE_READY`：提示开始背诵，说明可提示规则
* `RECITING`：用户背诵中（逐句对齐）
* `HINT_OFFER`：卡顿后询问是否提示
* `HINT_GIVEN`：给提示后回 `RECITING`
* `FINISHED`：完成表扬 + 询问下一首
* `EXIT`：超时/用户说退出/无有效输入

### 5.2 用户语音指令（NLU Intent）

* `RECITE_POEM`：我要背xx / 背xx / xx
* `SET_POEM`：确认就是这首
* `REJECT_POEM`：不是 / 换一首
* `DYNASTY_AUTHOR`：唐 李白 / 唐朝 李太白 / “南朝…谢灵运…”
* `START_RECITE`：我开始了 / 开始背
* `ASK_HINT`：提示一下 / 给点提示
* `EXIT_SESSION`：退出 / 不背了 / 停止
* `NEXT_POEM`：下一首 / 再来一首
* `REPEAT_PROMPT`：再说一遍

## 6. 核心算法口径（背诵容错、variant、口音）

### 6.1 诗名识别与抽取

**输入**：ASR 文本（可能含停顿词、重复、无意义词）
**步骤**：

1. 文本清洗：

   * 去掉口头禅：`嗯/啊/那个/就是/我想/我要/帮我/请`
   * 去掉书名号与多余符号
   * 合并重复片段（简单规则：连续重复 n-gram 删除）
2. 从清洗后文本中抽取候选：

   * 直接匹配诗库标题（精确）
   * 模糊匹配：编辑距离 / 拼音近似（可先做标题拼音索引）
3. 置信度策略：

   * 若唯一高置信度：进入 `WAIT_DYNASTY_AUTHOR`
   * 若多个接近：进入 `CONFIRM_POEM_CANDIDATE`，语音给 2~3 个选项让用户说“第一个/第二个/就这个”

### 6.2 朝代模糊匹配

* 用户说“南朝”“南北朝”“五代”“五代十国”均映射到同一 canonical dynasty（配置表驱动）。
* 当用户说出不确定朝代：

  * 若诗库中该标题跨多个朝代（极少）：提示用户确认
  * 否则可“弱校验”：只要作者匹配即可放行

### 6.3 作者别称匹配

* poem.author.aliases 支持字/号/别称
* 用户输入的作者字符串做：

  * 去空格/去“字/号/先生/诗仙”等修饰词
  * 与 primary + aliases 做包含/相似匹配

### 6.4 背诵对齐与判定（逐句）

**目标**：用户不必一口气背完整首；允许停顿、重复、回退一点点。
**建议实现**：句级对齐 + 字级容错评分

#### 6.4.1 句级对齐

* 当前目标句 `line[i]`
* 将用户最新一段 ASR 文本按标点/停顿分段（平台若不给分段，就用“短句长度阈值”切）
* 对每段尝试与 `line[i]` 的可接受集合匹配：

  * `variants` 文本集合（standard + historical variants）
  * 允许用户只说了前半句：部分匹配得分达到阈值即可推进或继续等待补全

#### 6.4.2 字级容错（简化但有效）

对比“用户文本 vs 目标文本（或某 variant）”：

* 先统一简繁（内部用简体对齐；展示随系统）
* 生成拼音序列（内置拼音表，不依赖网络）
* 计算综合得分：

  * 字符相等 +1
  * 同音（忽略声调） +0.9
  * 前后鼻音容忍：如 an/ang、en/eng、in/ing 视为近似 +0.8（由 config 控制）
  * 形近/常见错字：可选词表 +0.7
* 阈值建议：

  * 完整句通过：score >= 0.88 且长度覆盖 >= 0.85
  * 半句推进：score >= 0.90 且覆盖 >= 0.55（然后继续等用户补后半句）
* `tonePolicy=ignore` 时：拼音比较不计声调
* `toneRemind=true` 时：若字正确但声调不同，**不算错**，但在句通过后给一次轻提醒（“这个字读二声”）

### 6.5 历史认可 variant

* 每句 `variants[]` 里包含可替换说法
* 匹配时取最高得分 variant
* `remindCommonVariants=true` 时：若用户背的是 variant（非 standard），可以在通过后补一句“常见写法也会说成…（可关闭）”

## 7. 提示策略（卡顿检测与提示内容生成）

### 7.1 卡顿检测

* 在 `RECITING` 状态：

  * 若 `silence >= reciteSilenceAskHintSec` 且用户没有继续有效输入：

    * 进入 `HINT_OFFER`：问“要不要提示一下？”
  * 若 `HINT_OFFER` 中 `hintOfferWaitSec` 无回应：

    * 再提示一次“需要提示的话说‘提示一下’”
    * 仍无回应则回到 `RECITING` 或 `EXIT`（看总超时）

### 7.2 提示类型

对当前目标句 `line[i]`：

* **开头一个字**：返回 `text[0]`
* **下一个字**：基于用户已通过的前缀长度 k，返回 `text[k]`
* **大体意思**：返回 `meaning`（短句化，避免长解释）

提示选择：

* 若用户说“提示开头/提示下一个字/提示意思”则定向
* 否则默认优先“下一个字”，若用户还没开口就给“开头一个字”

## 8. 表扬与个性化（必须“每次不一样”）

### 8.1 表扬语料（内置模板 + 随机变量）

内置不少于 30 条模板，支持插槽：

* `{title}` `{author}` `{dynasty}` `{streak}`（连续背对句数）
* 例：

  * “太稳了！《{title}》你背得很顺。”
  * “这段节奏掌握得很好，继续保持～”
  * “厉害，几乎没有停顿！”
  * “背得真有感觉，像在给我讲故事一样。”

### 8.2 去重策略

* 维护最近 N 次表扬的 ID（N=5）
* 随机抽取时避开最近 N 次

## 9. 超时退出与安全边界

### 9.1 自动退出

* `WAIT_POEM_NAME` 若累计无有效 `RECITE_POEM` / `EXIT_SESSION` 超过 `noPoemIntentExitSec`：

  * 语音：“我先退出背诗模式啦，需要的话再点开始背诗。”
  * 状态 -> `EXIT` -> 回 `IDLE`

### 9.2 权限与失败兜底

* 麦克风权限未授予：弹系统权限页 + 语音/文字提示
* ASR/TTS 初始化失败：提示用户检查系统语音服务或网络（若使用系统云端 ASR）

## 10. 平台实现要点（Codex 按平台生成工程）

### 10.1 Android（Kotlin）

* ASR：优先系统 SpeechRecognizer（普通话 zh-CN），支持 partial results
* TTS：android.speech.tts.TextToSpeech，列出可用 voice（性别/风格有限，但可做多选）
* 音频焦点：AudioFocus 管理，TTS 播放时降低录音冲突
* UI：一个 Activity（开始/设置），开始后进入全屏“正在背诗”页但不要求触控

### 10.2 HarmonyOS / HarmonyOS NEXT（ArkTS）

* 使用系统语音识别/语音合成能力（按官方语音服务接口）
* ArkUI：主页 + 设置页 + Session 页
* 注意：Harmony NEXT 权限模型与能力调用与 HarmonyOS 有差异，平台层做适配封装，保持 Shared Core 不变

### 10.3 平台语音抽象接口（必须统一）

定义统一接口，平台实现：

```ts
interface ISpeechIO {
  startListening(params: { lang: "zh-CN"; partial: boolean }): void;
  stopListening(): void;
  onAsrResult(cb: (r: { text: string; isFinal: boolean; confidence?: number }) => void): void;

  speak(text: string, voiceId: string): Promise<void>;
  stopSpeak(): void;
  listVoices(): Promise<Array<{ id: string; name: string }>>;
}
```

## 12. 验收标准（必须可测试）

1. **开始背诗后无需触控**：除“开始背诗”点击外，能完成选诗、校验、提示、结束
2. **诗名识别鲁棒**：包含停顿/重复仍能提取诗名（至少命中内置诗库 Top 200）
3. **朝代模糊**：用户说“五代”能匹配“五代十国”类映射
4. **作者别称**：说“太白”能识别为李白（基于 aliases）
5. **variant 通过**：配置开启时，“豫章故郡/南昌故郡”任一说法通过同一句
6. **前后鼻音容忍**：开启容忍后，an/ang 等差异不判错
7. **声调提醒可选**：toneRemind 开关生效；默认不提醒
8. **卡顿提示**：停顿 6 秒会询问提示；提示后能继续
9. **表扬不重复**：连续 5 次完成不出现相同表扬语
10. **无输入自动退出**：25 秒无有效意图自动结束 Session

## 2. 朝代模糊映射：自动建立 + 用户可编辑

### 2.2 自动建立流程（Dynasty Knowledge Base）

**目标**：把各种说法（如“唐”“唐朝”“大唐”“李唐”）归到 canonical dynasty；并支持“用户说一个宽泛朝代”也能映射到 canonical（比如“南朝”覆盖宋齐梁陈）。

实现步骤（推荐）：

1. **抽取候选朝代词表**

   * 从本地诗库的 `dynasty` 字段收集
   * 从在线源返回的 `dynasty` 字段收集
2. **规范化（Normalization）**

   * 去掉后缀：`朝/代/时期/王朝`
   * 统一繁简
   * 常见前缀词处理：`大/后/前/南/北/西/东`（保留做特征）
3. **相似度聚类（Auto-Clustering）**

   * 用字符串相似（编辑距离、Jaccard n-gram）对 dynasty 名称聚类
   * 聚类后选频率最高的作为 canonical
4. **“宽泛朝代”自动关系**

   * 规则驱动（可扩展表，但不是手写死每个映射）：

     * 如果名称包含方向/前后（南/北/前/后/西/东），同时存在同根词（如“宋”与“南宋”“北宋”），则生成：

       * `宋 -> {北宋, 南宋}`
       * `南宋 -> {南宋}`（自映射）
     * 如果名称属于集合词（如用户说“南朝”），允许用户在 UI 中建立 “Group Dynasty = [宋齐梁陈]” 这种“群组映射”。首次遇到无法自动推断时，引导用户添加一次，后续复用。
5. **用户覆盖层（User Overrides）**

   * 用户可新增/编辑/删除映射：

     * `alias -> canonical`
     * `group -> [canonicals...]`
   * 用户修改优先级最高。

### A3. 规范化（Normalization）规则（必须一致）

```ts
const DYNASTY_SUFFIXES = ["朝", "代", "时期", "王朝"];
const TRIM_WORDS = ["大"]; // 可扩展：大唐/大宋 -> 唐/宋（谨慎）

export function normalizeDynasty(raw: string): string {
  let s = normalizeZh(raw);      // 简繁统一、全角半角、去空白
  s = stripPunct(s);
  s = s.replace(/\s+/g, "");

  for (const suf of DYNASTY_SUFFIXES) {
    if (s.endsWith(suf)) s = s.slice(0, -suf.length);
  }
  // “大唐”“大宋”这类前缀可去掉，但不建议过度
  for (const w of TRIM_WORDS) {
    if (s.startsWith(w) && s.length >= 2) s = s.slice(w.length);
  }
  return s;
}
```

> `normalizeZh/stripPunct`：你在 text_cleaner 里应已有；没有就放到 `utils/zh_normalize.ts`。

### A4. 自动建立 canonical：聚类 + 选中心

**原则**：能自动确定的（如 唐/唐朝）自动；不确定的宽泛关系（如 南朝=宋齐梁陈）不强行猜，交给用户 groups。

#### 阈值建议

* alias 聚类阈值：`sim >= 0.86`
* 同根词（宋/南宋/北宋）的“根”判断：`rootLen>=1` 且包含关系成立

#### 相似度（轻量可用）

用字符 bigram Jaccard + 编辑距离混合：

```ts
function sim(a: string, b: string): number {
  const j = jaccard2gram(a, b);           // 0~1
  const e = 1 - editDistance(a, b) / Math.max(a.length, b.length);
  return 0.6 * j + 0.4 * e;
}
```

#### 聚类伪代码

```ts
export interface BuildDynastyKBInput {
  observedDynasties: string[]; // 来自诗库/在线元数据的 raw dynasty
  existingKB?: DynastyKB;      // 若已有则增量更新
  now: number;
}

export function buildOrUpdateDynastyKB(input: BuildDynastyKBInput): DynastyKB {
  const now = input.now;
  const rawList = input.observedDynasties.filter(Boolean);
  const normList = rawList.map(normalizeDynasty).filter(s => s.length > 0);

  // 统计频次
  const freq = new Map<string, number>();
  for (const n of normList) freq.set(n, (freq.get(n) ?? 0) + 1);

  // 聚类：把 normName 聚成多个簇
  const names = Array.from(freq.keys());
  const clusters: string[][] = [];
  const used = new Set<string>();

  for (const name of names) {
    if (used.has(name)) continue;
    const cluster = [name];
    used.add(name);
    for (const other of names) {
      if (used.has(other)) continue;
      if (sim(name, other) >= 0.86) {
        cluster.push(other);
        used.add(other);
      }
    }
    clusters.push(cluster);
  }

  // 对每簇选 canonical：频次最高；平手取更短/更常见（可按 freq+len）
  const canonicals: DynastyCanonical[] = [];
  const aliasEdges: DynastyAlias[] = [];

  for (const cl of clusters) {
    const sorted = cl.slice().sort((a,b) => {
      const fa = freq.get(a) ?? 0, fb = freq.get(b) ?? 0;
      if (fb !== fa) return fb - fa;
      return a.length - b.length; // 更短优先
    });
    const canonicalNameNorm = sorted[0];
    const canonicalId = makeDynastyId(canonicalNameNorm); // "dyn_" + hash/slug

    const totalFreq = cl.reduce((sum, x) => sum + (freq.get(x) ?? 0), 0);
    canonicals.push({
      id: canonicalId,
      name: toPreferredDisplay(canonicalNameNorm), // 简体显示名（UI再转简繁）
      norm: canonicalNameNorm,
      freq: totalFreq
    });

    // cl 中其它都是 alias（含 canonical 自己也可不加 alias）
    for (const aliasNorm of cl) {
      if (aliasNorm === canonicalNameNorm) continue;
      aliasEdges.push({
        alias: aliasNorm,        // 这里存 norm 也可以，建议保留原始 raw 另存一份（可选）
        aliasNorm,
        canonicalId,
        source: "auto",
        updatedAt: now
      });
    }
  }

  // 生成“方向/前后/东西”等同根关系（可选自动 groups）
  // 例：宋 与 南宋/北宋 -> group alias "宋" 映射 [南宋,北宋,宋] 不一定需要；更推荐：alias->canonical 已解决
  // 我建议这里仅为 NLU 提供“根词提示”，不直接写入 groups，避免误导

  // 合并 existingKB：保留用户修改，auto 可更新
  const merged = mergeDynastyKB(input.existingKB, canonicals, aliasEdges, now);
  return merged;
}
```

### A5. 用户自定义添加/编辑 API（必须）

```ts
export function upsertDynastyAliasUser(
  kb: DynastyKB,
  aliasRaw: string,
  canonicalNameOrId: string,
  now: number
): DynastyKB {
  const aliasNorm = normalizeDynasty(aliasRaw);
  const canonicalId = resolveCanonicalId(kb, canonicalNameOrId) 
    ?? createCanonicalIfMissing(kb, canonicalNameOrId, now).id;

  // 删除同 aliasNorm 的 auto 记录，写入 user
  kb.aliases = kb.aliases.filter(a => a.aliasNorm !== aliasNorm || a.source === "user");
  const existing = kb.aliases.find(a => a.aliasNorm === aliasNorm && a.source === "user");
  if (existing) {
    existing.canonicalId = canonicalId;
    existing.updatedAt = now;
  } else {
    kb.aliases.push({ alias: aliasRaw, aliasNorm, canonicalId, source:"user", updatedAt: now });
  }
  kb.meta.updatedAt = now;
  kb.meta.version += 1;
  return kb;
}

export function upsertDynastyGroupUser(
  kb: DynastyKB,
  groupAliasRaw: string,
  canonicalIdsOrNames: string[],
  now: number

  const aliasNorm = normalizeDynasty(groupAliasRaw);
  const canonicalIds = canonicalIdsOrNames
    .map(x => resolveCanonicalId(kb, x) ?? createCanonicalIfMissing(kb, x, now).id);

  // 覆盖式写入 user group
  kb.groups = kb.groups.filter(g => g.aliasNorm !== aliasNorm || g.source === "user");
  const g = kb.groups.find(g => g.aliasNorm === aliasNorm && g.source === "user");
  if (g) { g.canonicalIds = unique(canonicalIds); g.updatedAt = now; }
  else kb.groups.push({ alias: groupAliasRaw, aliasNorm, canonicalIds: unique(canonicalIds), source:"user", updatedAt: now });

  kb.meta.updatedAt = now;
  kb.meta.version += 1;
  return kb;
}
```

```ts
export interface DynastyResolveResult {
  type: "canonical" | "group" | "unknown";
  canonicalIds: string[];      // canonical 或 group 展开
  matchedBy?: "exact" | "alias" | "group";
  confidence: number;          // 0~1
}

export function resolveDynastySpoken(kb: DynastyKB, spokenRaw: string): DynastyResolveResult {
  const n = normalizeDynasty(spokenRaw);
  if (!n) return { type:"unknown", canonicalIds:[], confidence:0 };

  // 1) 直接 canonical 命中
  const c = kb.canonicals.find(x => x.norm === n || x.name === n);
  if (c) return { type:"canonical", canonicalIds:[c.id], matchedBy:"exact", confidence:0.98 };

  // 2) alias 命中（user 优先）
  const aliases = kb.aliases
    .filter(a => a.aliasNorm === n)
    .sort((a,b) => (a.source==="user"?-1:1));
  if (aliases.length) return { type:"canonical", canonicalIds:[aliases[0].canonicalId], matchedBy:"alias", confidence:0.92 };

  // 3) group 命中（user 优先）
  const groups = kb.groups
    .filter(g => g.aliasNorm === n)
    .sort((a,b) => (a.source==="user"?-1:1));
  if (groups.length) return { type:"group", canonicalIds:groups[0].canonicalIds, matchedBy:"group", confidence:0.90 };

  // 4) 模糊：找最像的 canonical/alias（可选）
  let best: { id: string; score: number } | null = null;
  for (const c of kb.canonicals) {
    const sc = sim(n, c.norm);
    if (!best || sc > best.score) best = { id: c.id, score: sc };
  }
  if (best && best.score >= 0.86) return { type:"canonical", canonicalIds:[best.id], matchedBy:"alias", confidence:best.score };

  return { type:"unknown", canonicalIds:[], confidence:0.2 };
}
```

## 8. 背诵匹配与容错（Recite Matcher）

### 8.1 接受集合（每句）

`acceptedTexts = 标准句 + 本地variants + 在线variants（缓存）`

### 8.2 字级评分（轻量可落地）

* 简繁统一，去空白符
* 拼音对齐（忽略声调为默认策略）
* 得分组合：

  * 字符相等、同音（忽略声调）、前后鼻音近似等
* 通过阈值建议：

  * 完整句通过：`score>=0.88 && coverage>=0.85`
  * 半句推进：`score>=0.90 && coverage>=0.55`

### 8.3 声调策略

* 默认：忽略声调（不算错也不提醒）
* 可配：`toneRemind=true` 时，句通过后轻提醒正确声调（仍判通过）
